## 0. 先统一判断：当前项目的纠偏方向

### 0.1 当前问题的准确表述

当前项目的核心链路能跑通，成果包、Package 规范、下游契约、评测体系都比较完整；但项目表达和部分实现存在偏移：

```text
任务书理想形态：
标准 UIR + 目标 Schema + 元数据模板 + 映射规则 + 分段/打标参数
-> 转换智能体

-> Schema Router 路由到内置 5 类 schema/template
-> 候选抽取与映射服务中含大量领域特化规则
-> 标准成果包 + Review/Knowledge/Lineage/Evaluation 等平台能力
```

所以纠偏不是“删掉 Schema、规则、评测”，而是：

```text
1. 把 Schema / Template / MappingRules / ContentConfig 明确变成运行时输入配置。
2. 把内置 5 类 SchemaPack 降级为 examples，不再作为系统能力边界。
3. 把 Router 和 CandidateService 中的领域专用规则尽量配置化。
4. 增加一个最贴合任务书的标准 convert 接口。
5. 增加“无代码新增 SchemaPack”的演示和测试。
6. 重写 README、requirement_mapping、demo_script，让答辩主线回到课题 5。
```

### 0.2 纠偏后的项目一句话定位

最终项目定位必须统一为：

> 本项目实现课题 5“数据格式标准化转换智能体”。系统接收归一后的结构化中间表示 UIR、目标 Schema/元数据模板/映射规则和内容组织参数，完成字段映射、字段操作、Schema 校验、分段打标摘要和双形态成果包封装。仓库内置的 policy、meeting、procurement、contract、general 等 SchemaPack 只是示例配置与评测基准，不是系统能力边界。


建议结构：

```python
from typing import Any
from pydantic import BaseModel, Field

from app.schemas.target_schema import TargetSchema
from app.schemas.mapping_template import MappingTemplate
from app.schemas.uir import UIRDocument


class MetadataTemplateConfig(BaseModel):
    template_id: str
    schema_id: str
    version: str = "1.0.0"
    metadata_fields: list[dict[str, Any]] = Field(default_factory=list)


class ContentOrganizationConfig(BaseModel):
    chunking: dict[str, Any] = Field(default_factory=dict)
    tags: dict[str, Any] = Field(default_factory=dict)
    summary: dict[str, Any] = Field(default_factory=dict)
    keywords: dict[str, Any] = Field(default_factory=dict)
    source_links: dict[str, Any] = Field(default_factory=dict)


class Topic5ConvertRequest(BaseModel):
    uir: UIRDocument
    target_schema: TargetSchema
    mapping_template: MappingTemplate
    metadata_template: MetadataTemplateConfig | None = None
    content_organization: ContentOrganizationConfig = Field(default_factory=ContentOrganizationConfig)
    options: dict[str, Any] = Field(default_factory=dict)


class Topic5ConvertResponse(BaseModel):
    task_id: str
    status: str
    schema_id: str
    template_id: str
    content_json: dict[str, Any]
    content_markdown: str
    chunks: list[dict[str, Any]]
    mapping_report: dict[str, Any]
    transform_report: dict[str, Any]
    validation_report: dict[str, Any]
    content_organization_report: dict[str, Any]
    manifest: dict[str, Any] | None = None
    package_zip_path: str | None = None
```

注意：如果现有 `MappingTemplate` 不能直接表达 `mapping_rules.yaml` 的全部字段，先不要大改模型；第一阶段把 `mapping_rules.yaml` 转换成现有 `MappingTemplate` 支持的 aliases、regex_rules、enum_maps、defaults、transform_rules。负例规则可以先放入 `options["badcases"]` 或新增轻量字段。

### 5.4 标准转换服务

新建：

```text
backend/app/services/topic5_conversion_service.py
```

目标：复用现有转换组件，但绕过 catalog 强绑定。

核心实现逻辑：

```python
from app.schemas.topic5_convert import Topic5ConvertRequest, Topic5ConvertResponse
from app.services.candidate_service import CandidateService
from app.services.mapping_service import MappingService
from app.services.transform_service import TransformService
from app.services.canonical_service import CanonicalService
from app.services.render_service import RenderService, RenderedArtifacts
from app.services.validation_service import ValidationService
from app.services.chunk_organizer_service import ChunkOrganizerService
from app.services.package_service import PackageService
from app.services.schema_service import SchemaService
from app.services.template_service import TemplateService
from app.utils.ids import new_id


class Topic5ConversionService:
    def __init__(self, storage_root):
        self.storage_root = storage_root

    def convert(self, request: Topic5ConvertRequest, *, create_package: bool = False) -> Topic5ConvertResponse:
        task_id = new_id("topic5")
        doc_id = new_id("inline_doc")
        uir = request.uir
        schema = SchemaService().validate_schema(request.target_schema)
        template = TemplateService().validate_template(request.mapping_template, schema)

        options = dict(request.options or {})
        options["content_organization"] = request.content_organization.model_dump(mode="json")
        options.setdefault("enable_llm_fallback", False)
        options.setdefault("enable_lineage", False)
        options.setdefault("topic5_inline_config", True)

        candidates = CandidateService().extract_candidates(task_id, uir)
        mapping_report = MappingService().map_fields(
            task_id=task_id,
            uir=uir,
            schema=schema,
            template=template,
            candidates=candidates,
            options=options,
        )
        transform_result = TransformService().transform(task_id, uir, schema, template, mapping_report)
        execution_snapshot = {
            "task_id": task_id,
            "doc_id": doc_id,
            "schema_id": schema.schema_id,
            "schema_version": schema.version,
            "template_id": template.template_id,
            "template_version": template.version,
            "input_mode": "inline_topic5_config",
            "options": options,
        }
        canonical = CanonicalService().build_canonical(
            task_id=task_id,
            uir=uir,
            schema=schema,
            template=template,
            transform_result=transform_result,
            mapping_report=mapping_report,
            execution_snapshot=execution_snapshot,
        )
        rendered = RenderService().render(canonical, chunk_size=int(options.get("chunk_size", 1200)))
        preliminary_validation_report = ValidationService().validate(task_id, schema, rendered)
        organized_chunks, content_organization_report = ChunkOrganizerService().organize_chunks(
            chunks=rendered.chunks,
            canonical_model=canonical,
            schema=schema,
            mapping_report=mapping_report,
            validation_report=preliminary_validation_report,
            task_id=task_id,
            doc_id=doc_id,
            schema_id=schema.schema_id,
            template_id=template.template_id,
            template_version=template.version,
            options=options.get("content_organization"),
        )
        rendered = RenderedArtifacts(
            structured_json=rendered.structured_json,
            markdown=rendered.markdown,
            chunks=organized_chunks,
        )
        validation_report = ValidationService().validate(
            task_id,
            schema,
            rendered,
            require_content_organization=True,
        )

        manifest = None
        package_zip_path = None
        if create_package:
            package_result = PackageService(self.storage_root).create_package(
                task_id=task_id,
                doc_id=doc_id,
                schema=schema,
                template=template,
                canonical=canonical,
                rendered=rendered,
                mapping_report=mapping_report,
                transform_report=transform_result.report,
                validation_report=validation_report,
                content_organization_report=content_organization_report,
            )
            manifest = package_result.manifest.model_dump(mode="json")
            package_zip_path = package_result.metadata.zip_path

        return Topic5ConvertResponse(
            task_id=task_id,
            status="completed" if validation_report.passed else "review_required",
            schema_id=schema.schema_id,
            template_id=template.template_id,
            content_json=rendered.structured_json,
            content_markdown=rendered.markdown,
            chunks=rendered.chunks,
            mapping_report=mapping_report.model_dump(mode="json"),
            transform_report=transform_result.report,
            validation_report=validation_report.model_dump(mode="json"),
            content_organization_report=content_organization_report.model_dump(mode="json"),
            manifest=manifest,
            package_zip_path=package_zip_path,
        )
```

如果 `PackageService(self.storage_root)` 参数类型与当前实际实现不一致，以现有 `TaskExecutionService` 中调用方式为准：`PackageService(self.storage.root).create_package(...)`。

### 5.5 新增 API Router

新建：

```text
backend/app/api/v1/topic5.py
```

示例：

```python
from typing import Annotated
from fastapi import APIRouter, Body, Depends, HTTPException

from app.api.deps import get_storage_service
from app.schemas.topic5_convert import Topic5ConvertRequest, Topic5ConvertResponse
from app.services.storage_service import StorageService
from app.services.topic5_conversion_service import Topic5ConversionService

router = APIRouter(prefix="/topic5", tags=["topic5"])


def get_topic5_service(
    storage: Annotated[StorageService, Depends(get_storage_service)],
) -> Topic5ConversionService:
    return Topic5ConversionService(storage.root)


@router.post("/convert", response_model=Topic5ConvertResponse)
def convert_topic5(
    request: Annotated[Topic5ConvertRequest, Body()],
    service: Annotated[Topic5ConversionService, Depends(get_topic5_service)],
) -> Topic5ConvertResponse:
    try:
        return service.convert(request, create_package=False)
    except ValueError as exc:
        raise HTTPException(status_code=400, detail=str(exc)) from exc


@router.post("/convert/package", response_model=Topic5ConvertResponse)
def convert_topic5_package(
    request: Annotated[Topic5ConvertRequest, Body()],
    service: Annotated[Topic5ConversionService, Depends(get_topic5_service)],
) -> Topic5ConvertResponse:
    try:
        return service.convert(request, create_package=True)
```

## 8. MappingRules 配置化

### 8.1 目标

让运行时映射规则主要来自配置，而不是散落在代码中。

### 8.2 第一阶段必须实现的规则类型

```text
aliases：字段别名。
regex_rules：正则提取。
negative_pairs：禁止映射对。
transform_rules：字段转换规则。
defaults：默认值。
thresholds：置信度阈值。
candidate_hints：候选抽取提示。
```

### 8.3 MappingService 调整

当前 `FORBIDDEN_PAIRS` 是硬编码负例。第一阶段保留它，但新增配置负例入口。

在 `map_fields(... options)` 中读取：

```python
configured_forbidden_pairs = options.get("negative_pairs", [])
```

`_with_decision_context` 或 `_builtin_forbidden_reason` 前后增加配置负例判断：

```python
def _configured_forbidden_reason(self, source_name: str, target_field_id: str, negative_pairs: list[dict[str, Any]]) -> str | None:
    for item in negative_pairs:
        pattern = item.get("source_pattern")
        target = item.get("target_field_id")
        if target == target_field_id and pattern and re.search(pattern, source_name):
            return item.get("reason") or "configured_negative_pair"
    return None
```

要求：

```text
1. 配置负例命中时，不能 auto accept。
2. mapping_report 必须记录 negative_pair source = configured_rules。
3. badcase violations 仍应为 0。
```
