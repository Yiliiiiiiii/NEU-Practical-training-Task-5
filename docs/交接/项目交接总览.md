## 2. 产品定位

SchemaPack Agent 把标准化文档转换为受目标 Schema 约束的成果包，并提供：

- 确定性字段候选、映射、转换与校验；
- 人工 Review 与 Knowledge Pack 治理；
- manifest、hash、verifier 和 downstream consumer contracts；
- External UIR adapter、Schema Router、Draft Lab 和 Evaluation Center；
- SchemaPack-Lineage 字段/block/chunk/artifact 可信追溯。

核心链路保持：

```text
UIR / External UIR JSON
-> Adapter / Schema Router
-> Schema + Template Snapshot
-> Candidate Extraction
-> Mapping + Review / Knowledge
-> Transform + Canonical
-> Render + Content Organization
-> Validate
-> Manifest + ZIP
-> Verify + Consumer Contract
```

Lineage 是旁路记录层，不改变任何 mapping、transform、validation 或 package
结果。

## 3. 已实施能力

### 3.1 主转换链路

- 标准 UIR 导入、Document 查询、Task 创建/查询/显式执行。
- 5 类 catalog family：
  `contract_doc`、`general_doc`、`meeting_doc`、`policy_doc`、
  `procurement_doc`。
- exact、alias、regex、type、fuzzy mapping。
- confidence tier、mapping evidence、risk flag、review-required reason。
- badcase filter、required-field validation、canonical model、JSON/Markdown。
- heading-aware / parent-child content organization 与 enriched chunks。
- manifest、ZIP、strict package verifier。

### 3.2 External UIR 与路由

- 可插拔 adapter registry。
- block-list 与 section-tree 两种内置 dialect。
- adapter detection、trace evidence、External UIR convert/import/create-task API。
- Schema Router v2，多候选、证据、风险与人工确认。
- 18 个 fixtures；selection、validation、trace、router 均达到 1.0。
- DeepSeek 只提供 report-only suggestion，默认关闭且不会自动接受。

### 3.3 治理工作台

- Schema/Template Draft Lab：字段发现、风险检查、校验与导出。
- Review Workbench：分组、影响预览、批量安全与负知识。
- Knowledge Pack：candidate、draft/active/archived、diff/impact/rollback。
- Evaluation Center：dataset、run、metric、scorecard 与 regression gates。

### 3.4 下游集成

- Package 1.1。
- RAG corpus、training corpus、structured CSV consumer contracts。
- ZIP/directory consumer verifier。
- 统一 CLI、Python SDK、Adapter scaffold。

### 3.5 可选离线上游

- Docling/Unstructured 是惰性加载的离线入口。
- 可把带文本层的 PDF、DOCX、HTML 转为 External UIR。
- 不进入 backend runtime 默认依赖。
- 不提供生产级 raw-document upload API，也不实现 OCR。

### 3.6 SchemaPack-Lineage

- Lineage 1.0 schema、graph builder 和有界查询 service。
- 字段、chunk、artifact、External adapter、Review、Knowledge、manifest
  与 consumer contract 追溯。
- 五个 lineage API 和前端“可信链路”面板。
- 默认 `enable_lineage=true`、`strict_lineage=false`。
- evaluator 与四项 hard gates。

## 5. 运行方式

推荐一键启动：

```powershell
.\scripts\start_dev.ps1
```

手动启动 backend：

```powershell
Push-Location backend
.\.venv\Scripts\python.exe -m uvicorn app.main:app `
  --host 127.0.0.1 --port 8000
Pop-Location
```

手动启动 frontend：

```powershell
Push-Location frontend
npm.cmd run dev
Pop-Location
```

默认地址：

```text
Backend:  http://127.0.0.1:8000
Frontend: http://127.0.0.1:5173
```
